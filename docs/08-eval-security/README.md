# 08 Eval / Security / Monitoring

## 学习目标

让 AI 应用从 Demo 走向可用系统，具备评测、监控、安全和成本意识。

## Eval 评测

关注指标：

- 准确率
- 召回率
- 幻觉率
- 工具调用成功率
- 用户满意度
- 延迟
- 成本

## 测试方法

- Golden Dataset
- 回归测试
- A/B 测试
- LLM-as-Judge
- 人工评审
- 线上反馈闭环

## 监控内容

- Token 消耗
- 响应时间
- 错误率
- 工具调用失败率
- 用户问题分布
- 高风险请求

## 安全风险

- Prompt Injection
- Jailbreak
- 数据泄露
- 权限越权
- 工具滥用
- 命令注入
- SQL 注入
- 供应链风险

## 学习任务

1. 为一个 RAG 项目准备 50 条测试问题。
2. 记录每条问题的标准答案。
3. 评估模型回答是否正确。
4. 统计 token、耗时和失败率。
5. 设计敏感信息过滤策略。
6. 为高风险工具增加权限确认。

## 实战项目

做一个 `AI 应用评测面板`：

- 记录每次请求
- 记录模型回答
- 记录人工评分
- 展示成本和耗时
- 统计失败原因
