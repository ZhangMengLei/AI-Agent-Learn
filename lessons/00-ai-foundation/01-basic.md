# AI 基础讲解

## 1. AI、机器学习、深度学习、大模型是什么关系

可以把它们理解成一层一层包含的关系：

```text
AI 人工智能
└── 机器学习 Machine Learning
    └── 深度学习 Deep Learning
        └── 大模型 Foundation Model / LLM
```

- **AI**：让机器表现出类似智能的能力，例如识别图片、理解语言、规划任务。
- **机器学习**：不把规则全部手写死，而是让模型从数据中学习规律。
- **深度学习**：使用多层神经网络学习复杂模式。
- **大模型**：在大量数据上训练出来的通用模型，可以迁移到很多任务。
- **LLM**：Large Language Model，大语言模型，主要处理文本，也可以扩展到图片、音频、代码等多模态输入。

小白可以先不用理解训练细节，只要记住：

> LLM 不是一个“知识库表格”，而是一个根据上下文预测和生成内容的模型。

## 2. LLM 为什么能聊天、总结、写代码

LLM 的核心能力是根据前面的文本预测后面可能出现的文本。因为训练数据里包含大量文章、问答、代码、文档和对话，所以它学会了很多语言模式和代码模式。

当你输入：

```text
请总结下面这段话：...
```

模型会根据“总结任务”的模式生成摘要。当你输入：

```text
请解释这个 Python 函数：...
```

模型会根据代码解释的模式输出说明。

但它不是每次都在查事实数据库，所以可能出现：

- 说得很流畅但不正确。
- 编造不存在的 API、文件、参数。
- 忽略你没有明确说明的边界。

这就是为什么后续要学习 Prompt、RAG、Tool Use、Eval 和 Security。

## 3. Prompt 是什么

Prompt 是你给模型的任务说明书。好的 Prompt 通常包含：

1. 角色：你希望模型以什么身份回答。
2. 任务：要完成什么。
3. 背景：必要上下文。
4. 约束：不能做什么、必须遵守什么。
5. 输出格式：Markdown、JSON、表格等。
6. 验收标准：怎样算好答案。

差 Prompt：

```text
帮我写一下。
```

好 Prompt：

```text
你是资深 Python 老师。请解释下面函数的作用，面向刚学 Python 的学生。
输出包括：功能、输入、输出、关键步骤、可能的 bug。
如果信息不足，请明确说明不要猜测。
```

## 4. Token 和上下文窗口

模型不是直接按“字”理解内容，而是把文本切成 token。

可以简单理解为：

- 英文 token 常接近单词或词片段。
- 中文 token 可能是字、词或片段。
- 代码、标点、空格也可能占 token。

**上下文窗口**就是一次对话里模型能看到的最大 token 数。超过窗口的内容，模型可能看不到或需要压缩。

这会影响：

- 不能无限贴代码。
- 长文档需要切分、检索或总结。
- 多轮对话太长时，早期信息可能丢失。
- Agent 做复杂任务时要记录状态和中间结果。

## 5. Temperature 是什么

Temperature 可以粗略理解为“随机程度”：

- 低 temperature：更稳定、保守、适合代码、格式化、事实问答。
- 高 temperature：更多变化、适合创意写作、头脑风暴。

小白建议：

- 写代码、生成 JSON、做评测：低一些。
- 想标题、创意方案：可以高一些。

## 6. 幻觉是什么

幻觉是模型生成了看似合理但不真实的内容，例如：

- 编造不存在的函数。
- 引用不存在的文章。
- 说错配置项。
- 对不确定问题给出肯定答案。

降低幻觉的方法：

1. 给清楚上下文。
2. 要求“不确定就说不知道”。
3. 使用 RAG 提供可引用资料。
4. 使用工具查询真实数据。
5. 用测试和评测验证输出。

## 7. Chatbot、RAG、Agent、MCP 的区别

| 名称 | 解决什么问题 | 典型输入 | 典型输出 |
| --- | --- | --- | --- |
| Chatbot | 对话、解释、总结 | 用户问题 | 文本回答 |
| RAG | 基于资料回答 | 问题 + 检索文档 | 带引用的回答 |
| Tool Use | 调用外部能力 | 工具 schema + 参数 | 工具结果 |
| Agent | 多步骤完成任务 | 目标 + 工具 + 状态 | 行动、观察、最终结果 |
| MCP | 标准化接入工具和上下文 | tools/resources/prompts | 统一协议能力 |
| Claude Code | 在开发环境中协助工程任务 | 代码仓库 + 指令 | 计划、修改、测试、总结 |

一句话理解：

- Chatbot 偏回答。
- RAG 偏查资料后回答。
- Tool Use 偏调用具体能力。
- Agent 偏围绕目标多步执行。
- MCP 偏让工具接入更标准。
- Claude Code 偏把 Agent 用到真实开发流程中。

## 8. AI 开发和普通开发有什么不同

传统软件更像：

```text
输入固定 -> 规则固定 -> 输出可预测
```

AI 应用更像：

```text
输入变化大 -> 模型输出有不确定性 -> 需要评测、边界和兜底
```

所以 AI 工程更关注：

- Prompt 是否稳定。
- 上下文是否足够。
- 输出是否可验证。
- 工具调用是否有权限。
- 回答是否有引用。
- 是否能拒答。
- 是否有评测集和安全样例。

## 9. 小白常见误区

### 误区一：AI 什么都知道

更准确的说法：AI 能根据上下文生成答案，但不保证事实正确。

### 误区二：Prompt 是魔法咒语

Prompt 是任务说明，不是咒语。真正稳定的 AI 应用还需要数据、工具、测试和评测。

### 误区三：Agent 越自动越好

Agent 必须有边界。删除文件、发请求、改数据库、发邮件、push 代码都需要权限控制。

### 误区四：RAG 就是把文档塞给模型

RAG 需要切分、检索、重排、引用、拒答和评测。否则模型仍然可能乱答。

### 误区五：能跑通 demo 就能上线

上线还需要日志、监控、权限、成本控制、安全检查、回归评测和人工兜底。

## 10. 后续课程怎么学

推荐顺序：

1. 先学 Prompt：让模型听懂任务。
2. 再学 LLM API：知道应用如何调用模型。
3. 学 Tool Use：让模型能调用函数和工具。
4. 学 RAG：让模型基于资料回答。
5. 学 Agent：让模型多步骤完成任务。
6. 学 MCP：让工具和上下文接入标准化。
7. 学 Claude Code / Skill：把 Agent 用到开发工作流。
8. 学 Eval / Security：验证效果和安全边界。
