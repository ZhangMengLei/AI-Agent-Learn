# AI 基础实战项目：AI 学习导航卡片

## 项目背景

很多小白刚接触 AI 时会被术语淹没：Prompt、Agent、RAG、MCP、Embedding、Eval 都听过，但不知道分别解决什么问题。

本项目要求你做一份“AI 学习导航卡片”，帮助另一个初学者快速判断：我应该先学什么，我的需求适合用什么技术。

## 项目目标

产出一份 Markdown 文档或简单网页，包含：

1. AI 术语地图。
2. 常见需求到技术方案的选择表。
3. 小白常见误区说明。
4. 安全边界清单。
5. 后续学习路线。

## 用户故事

作为 AI 初学者，我希望：

- 能快速知道 AI、LLM、Agent、MCP 的关系。
- 能判断我的需求该用 Chatbot、RAG 还是 Agent。
- 能知道哪些内容不能直接交给 AI。
- 能知道下一步该学哪一章。

## 功能要求

### 1. 术语地图

至少包含：

- AI / ML / DL / LLM
- Prompt / Token / Context Window / Hallucination
- Embedding / RAG
- Tool Use / Agent / MCP
- Eval / Security

### 2. 方案选择表

示例：

| 需求 | 推荐方案 | 原因 | 风险 |
| --- | --- | --- | --- |
| 总结一段文章 | Prompt | 不需要外部数据 | 可能漏重点 |
| 基于公司制度问答 | RAG | 需要引用内部资料 | 权限和脱敏 |
| 自动修复测试失败 | Agent / Claude Code | 需要多步执行和验证 | 需要权限控制 |

### 3. 误区说明

至少说明 5 个误区：

- AI 不是数据库。
- Prompt 不是魔法。
- Agent 不是越自动越好。
- RAG 不只是塞文档。
- Demo 成功不等于可上线。

### 4. 安全边界

至少包含：

- 不提交真实 API Key。
- 不粘贴用户隐私。
- 高风险操作需要确认。
- 输出要验证。
- 生产操作要有回滚。

## 验收标准

- 初学者能在 10 分钟内读懂。
- 每个术语都有一句话解释。
- 每个推荐方案都有原因和风险。
- 没有真实密钥或敏感信息。
- 能链接到后续课程阶段。
