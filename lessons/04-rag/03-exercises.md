# RAG 练习

## 练习 1：文档切分

选择一篇 Markdown 文档，尝试设计切分策略。

回答：

1. 按标题切还是按长度切？
2. chunk size 设多少？
3. 是否需要 overlap？
4. 每个 chunk 需要保存哪些 metadata？

## 练习 2：设计 chunk 结构

为一段文档设计 JSON 结构。

必须包含：

- chunk_id
- document_id
- title
- content
- source
- section

## 练习 3：检索问题改写

把下面问题改写成更适合检索的形式：

```text
这个东西怎么用？
```

假设上下文是用户正在学习 MCP。

## 练习 4：判断检索质量

用户问题：

```text
Agent 和 Chatbot 有什么区别？
```

检索结果 A：介绍 Agent 的目标、工具调用和自主执行。

检索结果 B：介绍 Transformer 的注意力机制。

请判断哪个更相关，并说明原因。

## 练习 5：设计回答约束

写一个 Prompt，要求模型：

- 只基于检索资料回答
- 不确定时说不知道
- 回答后列出来源
- 不要使用资料外知识

## 练习 6：RAG 失败分析

分析下面问题可能由什么原因导致：

1. 检索不到相关内容
2. 检索到了但模型回答错误
3. 回答没有引用来源
4. 引用了错误来源
5. 回答太长或太短
