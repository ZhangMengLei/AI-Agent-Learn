# Tool Use 练习工程说明

本练习工程用于把前面的 Tool Use 概念落到可运行的小程序中。你会实现一个最小版“工具调用助手”，让模型能够在普通问答、计算、天气查询、读取学习笔记和高风险操作之间做选择。

重点不是接入真实服务，而是掌握工具定义、参数校验、工具执行、结果回传和权限确认的完整链路。

## 练习目标

完成后你应该能够：

1. 用结构化 Schema 描述工具名称、用途和参数。
2. 实现 `calculator`、`weather`、`mock notes` 三类本地工具。
3. 区分不需要工具的问题和必须调用工具的问题。
4. 对读取笔记、删除内容等操作加入权限分级。
5. 记录每次工具调用，便于调试和复盘。

## 推荐目录

建议在本目录下创建自己的实验代码，例如：

```text
03-exercises-lab/
  README.md
  src/
    main.py              # 命令行入口：接收用户输入并驱动一次对话
    tool_router.py       # 根据模型工具调用结果分发工具
    permissions.py       # 权限分级与确认逻辑
    tools/
      calculator.py      # 安全计算器工具
      weather.py         # Mock 天气工具
      notes.py           # Mock 学习笔记读取工具
    data/
      notes.json         # 本地笔记样例数据
    schemas/
      tools.json         # 工具定义 Schema
  tests/
    test_calculator.py
    test_permissions.py
```

如果你使用 JavaScript，也可以把 `src/main.py` 换成 `src/main.js`，其他结构保持类似即可。

## 工具设计要求

### 1. calculator：安全计算器

建议工具名：`calculate`

输入参数：

```json
{
  "expression": "12 * (3 + 4)"
}
```

要求：

- 只支持加、减、乘、除、括号、小数。
- 不允许直接执行用户输入代码。
- 计算失败时返回结构化错误，例如 `INVALID_EXPRESSION`。

### 2. weather：Mock 天气查询

建议工具名：`get_weather`

输入参数：

```json
{
  "city": "北京"
}
```

要求：

- 不调用真实天气 API。
- 用本地字典或 JSON 文件模拟 3 到 5 个城市天气。
- 查询不到城市时返回 `CITY_NOT_FOUND`。
- 提醒学生：真实项目中 API Key 应放在环境变量中，不要写进代码或文档。

### 3. mock notes：学习笔记读取

建议工具名：`read_note`

输入参数：

```json
{
  "title": "tool-use-basics"
}
```

要求：

- 从本地 `notes.json` 或固定字典读取内容。
- 只能读取预设笔记，不允许任意读文件路径。
- 返回笔记标题、摘要、正文片段。

### 4. 权限确认工具

建议先设计一个高风险工具，不一定真正执行，例如：

```text
delete_note(title)
```

要求：

- 默认不执行删除。
- 必须显示操作内容并要求用户确认。
- 用户输入 `yes` 后才允许继续。
- 练习阶段可以只打印“已模拟执行”，不要真的删除文件。

## 运行方式

以下命令是推荐形式，可按你的语言和包管理工具调整：

```bash
cd lessons/03-tool-use/03-exercises-lab
python -m venv .venv
source .venv/bin/activate
python src/main.py
```

如果暂时不接入真实大模型，可以先用规则模拟模型选择工具：

- 输入包含“计算”“多少”“+”“*”时调用 `calculate`。
- 输入包含“天气”时调用 `get_weather`。
- 输入包含“笔记”“学习记录”时调用 `read_note`。
- 输入包含“删除”时触发权限确认。

等流程跑通后，再把规则选择替换为大模型的 Tool Use / Function Calling。

## 建议练习顺序

1. 先实现所有工具的普通函数，不接入模型。
2. 为每个工具补齐 Schema 和参数校验。
3. 实现 `tool_router`，根据工具名调用对应函数。
4. 实现一轮对话流程：用户输入 → 工具选择 → 执行工具 → 输出答案。
5. 加入权限分级：低风险自动执行，高风险必须确认。
6. 加入日志：记录工具名、参数、结果、是否确认。

## 参考答案说明

本目录不强制提供唯一标准答案。参考答案应满足下面特征：

- 工具输入输出都是结构化对象，而不是随意拼接字符串。
- 计算器不会执行任意 Python / JavaScript 代码。
- Mock 天气和 Mock 笔记都可以离线运行。
- 高风险工具默认被拦截，不会自动执行。
- 错误信息对用户友好，对开发者可调试。

你可以用下面用例自测：

```text
计算 23 * 17 + 8
北京今天天气怎么样？
读取 tool-use-basics 笔记
删除 tool-use-basics 笔记
什么是工具调用？
```

## 验收标准

- 至少实现 3 个可调用工具：计算器、天气、笔记。
- 至少实现 1 个需要确认的高风险模拟工具。
- 普通知识问答不会误触发工具。
- 工具参数缺失或非法时能返回明确错误。
- 日志中能看到一次完整工具调用链。
