# Tool Use 实战项目：工具调用助手

## 项目目标

做一个可以根据用户问题自动选择工具的 AI 助手。

这个项目的重点不是工具本身有多复杂，而是理解模型、工具、程序执行和权限控制之间的关系。

## 功能要求

### 基础功能

- 支持至少 3 个工具
- 模型能判断是否需要调用工具
- 模型能生成工具参数
- 程序能执行工具并返回结果
- 模型能基于工具结果回答用户

### 推荐工具

```text
get_weather(city)       查询天气
calculate(expression)   计算数学表达式
read_note(title)        读取本地学习笔记
```

## 推荐目录

```text
tool-assistant/

  tools/

```

## 核心流程

```text
用户输入
  ↓
模型分析是否需要工具
  ↓
如果需要，生成工具调用
  ↓
程序校验参数
  ↓
程序执行工具
  ↓
工具结果回传模型
  ↓
模型生成最终回答
```

## 权限设计

低风险工具可以自动执行：

- 查询天气
- 数学计算

中风险工具可以记录日志后执行：

- 读取指定目录下的笔记

高风险工具必须人工确认：

- 删除文件
- 修改数据库
- 发送消息
- 执行命令

## 验收标准

- 能正确区分普通问答和工具问题。
- 能调用至少 3 个工具。
- 工具参数是结构化的。
- 工具失败时能给出友好提示。
- 高风险工具不会自动执行。
- 工具调用日志可追踪。
