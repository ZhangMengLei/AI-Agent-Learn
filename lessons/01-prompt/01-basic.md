# Prompt 基础

## Prompt 是什么

Prompt 可以理解为给 AI 的任务说明书。

写得越清楚，AI 输出越稳定；写得越模糊，AI 越容易跑偏、编造或输出不可用的结果。

## 好 Prompt 的基本结构

推荐先使用这个结构：

```text
角色：你要让 AI 扮演什么角色。
任务：你希望 AI 完成什么事情。
背景：给 AI 哪些上下文信息。
约束：输出长度、风格、限制条件。
格式：用 Markdown、JSON、表格还是其他格式输出。
示例：如果有固定风格，给一个参考例子。
```

简化记忆：

```text
角色 + 任务 + 背景 + 约束 + 输出格式
```

## 普通 Prompt 与优化 Prompt 对比

### 普通写法

```text
帮我总结这篇文章。
```

问题：

- 不知道总结多长
- 不知道给谁看
- 不知道要不要保留细节
- 不知道输出什么格式
- 不知道重点关注什么

### 优化写法

```text
你是一名擅长阅读技术文章的 AI 助手。

请帮我总结下面这篇文章。

要求：
1. 用中文回答
2. 先用 3 句话概括核心内容
3. 再列出 5 个关键要点
4. 最后给出一个适合初学者的理解建议
5. 不要编造文章中没有的信息

文章内容：
{{这里粘贴文章}}
```

## Prompt 的 5 个核心技巧

### 1. 明确角色

```text
你是一名有 10 年经验的增长营销专家，请帮我写一段面向程序员的 AI 工具推广文案。
```

### 2. 明确任务

```text
请分析这个产品需求，指出：
1. 核心目标
2. 用户场景
3. 可能遗漏的边界情况
4. 技术实现风险
```

### 3. 明确输出格式

```text
请用 Markdown 表格输出，包含三列：问题、原因、解决方案。
```

### 4. 给约束

```text
要求：
1. 不要编造不存在的信息
2. 如果不确定，请说“不确定”
3. 回答控制在 300 字以内
4. 面向初学者解释
```

### 5. 给示例

```text
请按照下面风格生成 3 条标题：

示例：
输入：AI 写代码
输出：AI 编程助手：从 Copilot 到 Agent 的演进

现在请为下面主题生成标题：
主题：MCP 入门
```

## 什么时候需要写更详细的 Prompt

适合写详细 Prompt 的场景：

- 要求输出稳定格式
- 输出要给程序继续处理
- 任务复杂，需要分步骤完成
- 有明确角色、风格或边界要求
- 需要避免模型编造

简单随意提问适合探索；正式任务要写成任务说明书。

## 一句话总结

Prompt 的本质不是提问，而是给 AI 写清楚任务说明书。
