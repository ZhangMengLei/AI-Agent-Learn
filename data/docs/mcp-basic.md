# MCP 基础

MCP（Model Context Protocol）是一种把模型应用和外部工具、数据源、上下文能力连接起来的协议思路。它的核心价值是让工具接入方式更加标准化，减少每个 Agent 系统重复造轮子。

## MCP 解决什么问题

在没有统一协议时，不同工具通常有不同的调用方式、认证方式和返回格式。Agent 想接入文件系统、数据库、搜索服务、内部平台时，需要写大量适配代码。

MCP 通过服务端暴露资源和工具，客户端按协议发现、调用和读取结果，让模型应用可以用更统一的方式访问外部能力。

## 常见组成

- MCP Client：通常集成在模型应用、IDE、CLI Agent 中。
- MCP Server：负责把某类能力暴露为资源、工具或提示模板。
- Tools：可执行动作，例如查询文档、创建任务、读取文件。
- Resources：可读取上下文，例如文档内容、配置、数据记录。
- Prompts：可复用的提示模板。

## 工程化关注点

- 工具描述要清晰，参数 schema 要稳定。
- 高风险工具需要权限控制和审计日志。
- 返回结果应尽量结构化，便于模型继续推理。
- 不要在工具描述、日志或样例中暴露真实密钥。

## 练习建议

选择一个很小的能力，例如“读取课程目录”或“查询样例知识库文档”，思考如何把它设计成 MCP 工具：工具名、参数、返回值和错误处理分别是什么。
