# Prompt 基础

Prompt 是人与大模型协作时的任务说明书。一个好的 Prompt 不只是“问一句话”，而是把目标、角色、输入、约束、输出格式和质量标准都说清楚。

## 为什么要结构化 Prompt

在课程练习中，建议把 Prompt 拆成以下部分：

1. 角色：模型应该以什么身份思考，例如“你是经验丰富的 AI 教学讲师”。
2. 任务：本次要完成什么，例如“把一段需求拆成可执行步骤”。
3. 上下文：模型需要知道哪些背景，例如业务场景、用户画像、已有代码结构。
4. 约束：不能做什么，必须遵守什么，例如“不要编造 API Key”“输出中文”。
5. 输出格式：希望结果是列表、表格、JSON，还是可运行代码。
6. 验收标准：如何判断回答可用，例如“包含风险、测试建议、下一步”。

## 示例模板

```text
你是{角色}。

背景：
{上下文}

任务：
{要完成的目标}

约束：
- {约束 1}
- {约束 2}

输出格式：
{格式要求}

验收标准：
- {标准 1}
- {标准 2}
```

## 练习建议

把同一个任务分别写成“随口提问”和“结构化 Prompt”，比较两次回答在准确性、完整性和可执行性上的差异。
