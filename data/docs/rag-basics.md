# RAG 入门

## 基本流程

RAG 是检索增强生成。它先从知识库中检索和问题相关的片段，再把这些片段作为上下文交给生成模块回答。

一个典型 RAG 流程包括文档加载、文档切分、索引构建、用户提问、相关片段检索、基于上下文生成答案、返回答案和引用来源。

## 为什么要切分文档

长文档直接检索容易不精确，也会浪费上下文空间。Chunking 会把文档切成更小的片段，让检索结果更聚焦。

常见切分方式包括按标题切分、按段落切分、固定长度切分，以及带重叠窗口的切分。

## 引用的重要性

RAG 答案应该带有来源引用。引用能帮助用户确认答案来自哪个文档和章节，也方便排查检索是否命中了正确内容。
