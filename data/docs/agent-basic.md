# Agent 基础

Agent 可以理解为围绕目标持续行动的系统。它通常不只生成一次文本，而是会在目标、计划、工具、观察结果之间循环，直到任务完成或触发停止条件。

## 一个最小 Agent 循环

1. 接收目标：明确用户想完成什么。
2. 制定计划：把目标拆成若干可执行步骤。
3. 选择工具：判断是否需要搜索、读写文件、调用 API、运行测试等。
4. 执行动作：调用工具或生成中间结果。
5. 观察结果：读取工具返回、错误信息或环境变化。
6. 调整策略：根据观察结果继续下一步或结束。

## 工程化关注点

- 边界：Agent 能做什么，不能做什么。
- 权限：哪些操作需要用户确认，哪些操作只读即可执行。
- 记忆：哪些上下文需要保留，哪些应该丢弃。
- 日志：如何追踪每一次工具调用和决策。
- 评测：如何判断 Agent 是否真的完成任务。
- 安全：如何避免越权操作、泄露密钥、执行危险命令。

## 练习建议

设计一个“学习计划 Agent”：输入学习目标和可用时间，输出计划；如果缺少信息，先提问；如果信息足够，再生成分阶段任务和检查点。
