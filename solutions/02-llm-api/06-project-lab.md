# LLM API 与 SDK 项目 Lab 参考路线

对应项目 Lab：[lessons/02-llm-api/06-project-lab/README.md](../../lessons/02-llm-api/06-project-lab/README.md)

## 建议实现步骤

1. 建立最小目录结构，只包含必要文件。
2. 写一个固定输入样例，先跑通主流程。
3. 拆出核心模块：client 层封装模型调用, conversation 层管理历史, logger 记录 token/耗时/错误。
4. 增加日志或中间结果输出，方便教学时观察。
5. 增加失败样例，检查边界行为。
6. 写运行说明和测试方式。

## 如何测试

- 使用课程提供的样例数据或自己构造的最小输入。
- 至少测试一个正常样例和一个边界样例。
- 如果项目涉及工具、文件、网络或外部系统，先用 mock 数据验证。
- 不在测试中依赖真实 API Key。

## 复盘问题

- 输入是否清晰？
- 输出是否稳定？
- 失败时是否能解释原因？
- 是否有权限边界？
- 是否能让另一个学习者按 README 跑起来？

## 和参考实现的关系

当前阶段没有单独的可运行参考实现，建议先完成课程中的目录和文档设计，再逐步补代码。
