## 练习 1：文章总结 Prompt

### 参考答案

```text
你是一名面向初学者的中文学习助手。请阅读我提供的文章，只基于文章内容总结，不要补充文章外信息。

请按固定格式输出：
一句话总结：用一句话概括文章核心观点。
核心要点：用 3-5 条 bullet 总结文章中的关键事实或观点。
适合谁看：说明这篇文章适合哪类初学者阅读。
我的建议：给初学者 1-2 条可执行学习建议，建议必须来自文章内容。

文章：
{{article}}
```

### 为什么这样答

这个 Prompt 明确了角色、语言、信息边界、读者对象和输出格式，因此比“帮我总结一下”更稳定。"只基于文章内容"能降低模型补充外部信息的概率。

### 常见错误

- 没有限定中文和初学者视角，导致回答风格不稳定。
- 只写格式不写“不编造”，模型可能加入文章外建议。
- 没有用占位符标出文章位置，复用时容易混乱。
## 练习 2：代码解释 Prompt

### 参考答案

```text
你是一名耐心的编程老师。请用中文给新手解释下面这段代码，不要假设读者熟悉高级术语。

请按以下结构回答：
1. 整体作用：这段代码解决什么问题。
2. 关键逻辑：按执行顺序解释主要步骤。
3. 易错点：指出新手最容易误解或改错的地方。
4. 简化版本：给出更短或更容易理解的伪代码/简化代码。

代码：
{{code}}
```

### 为什么这样答

解释代码时最重要的是先建立全局理解，再进入细节。固定四段结构能避免模型只逐行翻译代码，也能让新手知道应该关注哪些风险点。

### 常见错误

- 只要求“解释代码”，没有指定读者水平。
- 没有要求简化版本，学习者不容易迁移理解。
- 没有要求易错点，缺少实践价值。
## 练习 3：JSON 信息提取 Prompt

### 参考答案

```text
从用户输入中提取结构化信息。严格只输出一个合法 JSON 对象，不要输出 Markdown、解释或额外文本。

JSON 字段固定如下：
{
  "name": string | null,
  "job": string | null,
  "goal": string | null,
  "keywords": string[]
}

规则：
- 输入中没有出现的字段填 null。
- keywords 必须是数组；没有关键词时输出 []。
- 不要猜测用户没有明确表达的信息。

用户输入：
{{input}}
```

### 为什么这样答

这类任务的关键是约束输出边界。明确字段类型、缺失值规则、禁止解释，可以让结果更容易被程序解析。

### 常见错误

- 使用空字符串表示缺失，但题目要求 `null`。
- keywords 输出成字符串而不是数组。
- 在 JSON 前后添加说明文字，导致解析失败。
## 练习 4：对比实验

### 参考答案

同一任务示例：让模型解释“什么是 Agent”。

**Prompt 1：简单 Prompt**

```text
解释一下什么是 Agent。
```

**Prompt 2：带角色和任务**

```text
你是一名 AI 入门课老师。请用中文解释什么是 Agent，让没有 AI 背景的开发者也能理解。
```

**Prompt 3：带角色、任务、约束和格式**

```text
你是一名 AI 入门课老师。请用中文解释什么是 Agent，面向没有 AI 背景但会编程的开发者。

要求：
- 不使用复杂数学术语。
- 用 Chatbot 做对比。
- 给出一个工程例子。
- 不夸大 Agent 能力。

输出格式：
1. 一句话定义
2. 和 Chatbot 的区别
3. 工程例子
4. 常见误区
```

### 为什么这样答

Prompt 1 容易得到泛泛解释；Prompt 2 改善语气和受众；Prompt 3 同时约束内容、边界和格式，最适合教学场景。

### 常见错误

- 比较时只看回答长短，不看准确性和可复用性。
- 没有记录同一任务下的差异，实验不可复盘。
- 把 Prompt 写得过度复杂，反而掩盖核心目标。
