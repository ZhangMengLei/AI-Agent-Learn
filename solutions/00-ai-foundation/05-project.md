# 00 AI 基础项目参考讲解

## 项目核心

“AI 学习导航卡片”不是为了展示复杂技术，而是帮助小白建立判断能力：

- 这个需求是不是只需要 Prompt？
- 是否需要接入资料，也就是 RAG？
- 是否需要调用工具？
- 是否需要 Agent 多步执行？
- 是否需要 MCP 做标准化接入？
- 是否需要评测和安全边界？

## 推荐结构

```markdown
# AI 学习导航卡片

## 1. 我该先学什么

## 2. 术语地图

## 3. 需求到方案选择表

## 4. 常见误区

## 5. 安全边界

## 6. 后续学习路线
```

## 评分标准

| 维度 | 满分 | 说明 |
| --- | --- | --- |
| 清晰度 | 25 | 小白是否能读懂 |
| 准确性 | 25 | 概念是否没有明显混淆 |
| 场景判断 | 20 | 是否能把需求映射到合适能力 |
| 安全意识 | 20 | 是否说明密钥、隐私、权限风险 |
| 可继续学习 | 10 | 是否链接到后续课程 |

## 常见问题

### 1. 表格太抽象

给每个术语配一个具体开发例子。

### 2. 所有需求都推荐 Agent

这是错误倾向。Agent 适合多步执行，不适合所有简单问答。

### 3. 忽略 Eval

只要 AI 输出会影响用户或业务，就应该考虑评测和安全边界。
