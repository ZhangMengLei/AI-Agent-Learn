# 00 AI 基础练习参考答案

## 练习 1：概念关系图

参考关系：

```text
AI
└── Machine Learning
    └── Deep Learning
        └── Foundation Model / LLM

LLM 应用
├── Prompt：描述任务
├── RAG：接入外部资料
├── Tool Use：调用函数或工具
├── Agent：围绕目标多步执行
├── MCP：标准化接入工具、资源和提示模板
└── Eval / Security：评测质量和控制风险
```

重点不是画得唯一正确，而是能说明：LLM 是 AI 的一种重要模型能力，RAG、Tool、Agent、MCP 是围绕 LLM 构建应用的方法。

## 练习 2：区分 Chatbot、RAG、Agent

1. 解释 Python 中 list 和 tuple：Chatbot / Prompt 即可，主要是概念解释。
2. 根据公司制度回答员工问题：RAG，因为需要基于资料并给出处。
3. 检查 PR、运行测试、失败后修复：Agent / Claude Code，因为需要多步骤执行和验证。
4. 用户输入转固定 JSON：Prompt / LLM API，重点是结构化输出。
5. 每天读取构建结果并创建 issue：Agent + Tool Use，需要定时、读取外部系统、创建任务。

## 练习 3：识别幻觉风险

风险词：

- “一定支持”
- “auto_magic_config”
- “解决所有问题”

验证方式：

1. 查官方文档或源码。
2. 在项目中搜索配置项。
3. 写最小复现。
4. 查看测试或 issue。

更谨慎 Prompt：

```text
请检查这个库是否支持 auto_magic_config。不要猜测。
如果无法从我提供的文档或代码确认，请回答“无法确认”，并告诉我应该查哪里。
```

## 练习 4：设计一个好 Prompt

参考答案：

```text
你是 Python 入门老师。请解释下面代码，面向刚学函数和列表的学生。

背景：我想理解它的输入、输出和关键逻辑。
代码：<代码>

请输出：
1. 这段代码解决什么问题
2. 输入和输出
3. 每一步做了什么
4. 可能的边界情况
5. 如果有不确定点，请明确写“无法从代码确认”

验收标准：解释必须基于代码本身，不编造外部上下文。
```

## 练习 5：AI 使用安全判断

1. 不安全。`.env` 可能包含真实密钥。
2. 相对安全。前提是日志已脱敏。
3. 安全。测试草稿仍需人工检查。
4. 不安全。`git push --force` 是高风险操作，必须人工确认。
5. 安全。公开 README 可以用于生成文档，但要检查是否编造功能。

## 练习 6：学习路线自评

没有唯一答案。参考判断：

- 如果只是总结、分类、格式转换：先用 Prompt。
- 如果要集成到程序：需要 LLM API。
- 如果要基于私有文档回答：需要 RAG。
- 如果要调用外部系统：需要 Tool Use。
- 如果要多步骤完成目标：需要 Agent。
- 如果要给多个客户端标准化接入工具：考虑 MCP。
- 如果要上线或长期维护：需要 Eval / Security。
