# 00 AI 基础项目 Lab 参考讲解

## 最小实现路线

### 第一步：写 README

说明目标读者：

```markdown
这份导航面向刚接触 AI 开发的人，帮助你判断一个需求应该用 Prompt、RAG、Agent 还是 MCP。
```

### 第二步：写 glossary

建议至少写 12 个术语。每个术语控制在 100-200 字，避免复制百科式解释。

### 第三步：写 decision-table

推荐从自己的开发场景里选例子：

- 解释报错。
- 生成单元测试。
- 写 PR 描述。
- 公司文档问答。
- 自动排查 CI 失败。

### 第四步：写 safety-checklist

分三段最清楚：

1. 输入给 AI 前。
2. AI 输出后。
3. 提交或上线前。

## 进阶扩展

如果想做成网页，可以用静态 HTML：

- 左侧是术语列表。
- 中间是方案选择表。
- 右侧是安全清单。

如果想做成小工具，可以让用户回答几个问题：

1. 是否需要外部资料？
2. 是否需要调用工具？
3. 是否需要多步执行？
4. 是否涉及敏感数据？

然后输出推荐方案。

## 验收方式

找一个没学过本课程的人阅读 10 分钟，问 TA：

- Prompt 和 RAG 有什么区别？
- Agent 什么时候才需要？
- 什么信息不能发给 AI？
- 下一步应该学哪一章？

如果 TA 能回答，说明项目有效。
