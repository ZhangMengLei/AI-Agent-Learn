# Tool Use / Function Calling 练习 Lab 参考讲解

对应 Lab：[lessons/03-tool-use/04-exercises-lab/README.md](../../lessons/03-tool-use/04-exercises-lab/README.md)

## 参考目标

Tool Use 练习工程 的目标不是一次做成完整产品，而是把本阶段的核心概念变成一个能检查、能复盘的小实验。

## 推荐流程

1. 先写清楚实验输入：你要处理的问题、样例文本、样例请求或样例工具。
2. 再写清楚实验输出：Prompt 输出、API 日志、工具结果、检索结果、报告或评分。
3. 按最小路径实现：参考 `implementations/03-tool-assistant/`，实现工具注册、参数校验、权限确认和 tool log。
4. 记录一次成功样例和一次失败样例。
5. 用 checkpoint 或本目录答案检查理解是否到位。

## 验收标准

- 有清晰的输入和输出。
- 能复现实验结果。
- 能说明核心步骤为什么这样设计。
- 重点检查写工具是否需要确认、工具结果是否被当作不可信数据。
- 没有真实密钥、隐私数据或不可控高风险操作。

## 常见错误

- 只写最终答案，不记录实验过程。
- 没有失败样例，无法判断方案边界。
- 把模型输出当作一定正确，没有人工检查或规则校验。

## 可选扩展

- 增加 2-3 个边界样例。
- 把实验记录整理成 Markdown 表格。
- 为关键逻辑增加最小测试或自查清单。
