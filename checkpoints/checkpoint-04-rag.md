# Checkpoint 04：RAG 知识库问答

## 概念自查

1. RAG 为什么需要“检索”和“生成”两个阶段？
2. 文档切分的粒度会如何影响召回率和回答质量？
3. embedding、向量库、相似度检索分别是什么？
4. 召回、重排、引用来源在 RAG 中分别解决什么问题？
5. 为什么 RAG 仍然可能产生幻觉？

## 实践自查

1. 能否把 `data/docs/` 中的 Markdown 文档切分成 chunk？
2. 能否为每个 chunk 保留来源、标题、段落等元数据？
3. 能否针对一个问题召回最相关的文档片段？
4. 能否要求回答必须引用来源，找不到答案时说明不知道？
5. 能否用 `data/eval/golden.jsonl` 做基础效果检查？

## 工程自查

1. 是否区分了原始文档、索引文件和生成输出？
2. 是否有重新构建索引的命令或脚本？
3. 是否记录了文档版本、chunk 参数和 embedding 模型？
4. 是否能排查“召回不到”和“召回到了但答错了”两类问题？
5. 是否避免把敏感文档加入公开样例数据？
