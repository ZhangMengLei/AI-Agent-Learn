# Checkpoint 07：Claude Code / CLI Agent

## 概念自查

1. CLI Agent 相比普通聊天机器人多了哪些工程能力？
2. Claude Code 中工作目录、权限、工具调用对任务结果有什么影响？
3. Skill、Hook、配置文件分别适合解决什么问题？
4. 为什么处理代码任务时要先理解仓库结构和现有约定？
5. 为什么不应让 Agent 默认提交代码或 push 到远端？

## 实践自查

1. 能否让 CLI Agent 先读取相关文件，再进行最小必要修改？
2. 能否明确告诉 Agent 不要提交、不 push、不写真实密钥？
3. 能否把一个大任务拆成可检查的小步骤？
4. 能否要求 Agent 在完成后报告改动文件、风险和建议？
5. 能否识别 Agent 输出中哪些是事实，哪些是推测？

## 工程自查

1. 是否在隔离分支或 worktree 中完成高风险修改？
2. 是否查看了 git diff，确认没有误改无关文件？
3. 是否遵守项目的格式化、测试和提交规范？
4. 是否避免把本地配置、密钥和临时文件加入版本管理？
5. 是否在自动化 Hook 中控制了权限和副作用？
