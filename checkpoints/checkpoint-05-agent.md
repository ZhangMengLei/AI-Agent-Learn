# Checkpoint 05：Agent 架构

## 概念自查

1. Agent 和一次性 LLM 调用的本质区别是什么？
2. 目标、计划、行动、观察、反思分别在 Agent 循环中承担什么职责？
3. 什么任务适合做成 Agent，什么任务不适合？
4. Agent 的停止条件应该如何设计？
5. 记忆、工具、环境反馈对 Agent 有什么影响？

## 实践自查

1. 能否把一个复杂任务拆成可观察的多个步骤？
2. 能否设计 Agent 的系统提示和工具列表？
3. 能否让 Agent 在工具失败时调整计划，而不是无限重试？
4. 能否限制 Agent 的最大步骤数、最大成本和最大运行时间？
5. 能否输出任务执行摘要，说明做了什么、没做什么、下一步是什么？

## 工程自查

1. 是否有清晰的权限边界和危险操作确认机制？
2. 是否记录了每一步决策、工具调用和观察结果？
3. 是否能复现一次 Agent 执行过程？
4. 是否为关键任务设计了回滚或人工接管方式？
5. 是否能用评测集比较不同 Agent 策略的效果？
