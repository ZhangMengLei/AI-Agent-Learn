# Checkpoint 02：LLM API 与 SDK

## 概念自查

1. 一次 LLM API 调用通常包含哪些输入参数？
2. model、messages、temperature、max_tokens 分别影响什么？
3. 同步调用、流式调用和批处理调用分别适合什么场景？
4. token 用量、上下文长度和成本之间有什么关系？
5. 为什么生产环境中不能把 API Key 写死在代码里？

## 实践自查

1. 能否从环境变量读取 API Key？
2. 能否处理请求超时、限流、网络失败和模型错误？
3. 能否把模型响应解析成业务需要的数据结构？
4. 能否记录必要日志，同时避免记录敏感信息？
5. 能否为 API 调用封装一个可替换 provider 的函数或类？

## 工程自查

1. 是否有 `.env.example` 说明需要哪些配置？
2. 是否设置了合理的超时、重试和错误提示？
3. 是否能在没有真实 API Key 的情况下运行基础测试？
4. 是否区分了开发、测试和生产配置？
5. 是否监控了调用量、失败率和成本？
