# Checkpoint 06：MCP

## 概念自查

1. MCP 想解决模型应用接入外部能力时的什么问题？
2. MCP Client 和 MCP Server 分别是什么？
3. Tools、Resources、Prompts 的区别是什么？
4. 为什么 MCP 工具的参数 schema 和描述非常重要？
5. MCP 与普通 HTTP API 封装有什么联系和区别？

## 实践自查

1. 能否为一个只读查询能力设计 MCP 工具？
2. 能否定义工具输入参数、返回结构和错误结构？
3. 能否判断一个能力更适合作为 Tool 还是 Resource？
4. 能否描述 MCP Server 如何处理鉴权和权限边界？
5. 能否用最小样例解释 Client 如何发现并调用工具？

## 工程自查

1. 是否避免在工具描述和日志中暴露真实密钥？
2. 是否为高风险工具提供权限控制和审计日志？
3. 是否能处理工具版本变更和兼容性问题？
4. 是否有本地调试 MCP Server 的方法？
5. 是否有办法观测工具调用成功率、耗时和错误原因？
