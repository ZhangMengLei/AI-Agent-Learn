# Checkpoint 08：Eval / Security

## 概念自查

1. 为什么 AI 应用需要评测，而不能只靠人工感觉？
2. 黄金集、离线评测、在线监控分别适合什么场景？
3. 准确性、鲁棒性、延迟、成本、安全性分别如何观察？
4. Prompt 注入、数据泄露、越权工具调用分别是什么风险？
5. 安全策略应该只写在 Prompt 里，还是也要在代码和权限层实现？

## 实践自查

1. 能否为一个 Demo 准备 5 到 10 条黄金集样例？
2. 能否定义“答对”“部分答对”“答错”的判定标准？
3. 能否记录每次运行使用的模型、Prompt 版本和数据版本？
4. 能否设计一个恶意输入，测试系统是否会泄露敏感信息？
5. 能否对高风险工具调用增加确认、白名单或沙箱？

## 工程自查

1. 是否有评测命令或脚本可以重复运行？
2. 是否把评测数据、运行结果和配置分开管理？
3. 是否检查 `.env`、日志和样例数据中没有真实密钥？
4. 是否对失败样例进行分类，并形成改进计划？
5. 是否在上线前确认权限、审计、回滚和人工接管机制？
